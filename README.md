# C Elevator State Machine

## 项目简介

这是一个使用 C 语言实现的单电梯状态机模拟项目。

项目以“电梯运行控制”为场景，重点练习 C 语言中的结构体、枚举、函数拆分、头文件与源文件分离、状态机、事件处理、请求调度和异常安全机制。

本项目不是为了模拟真实商用电梯的全部复杂逻辑，而是通过一个接近现实的控制系统案例，训练如何把现实流程抽象成 C 语言程序。

## 当前项目阶段

本项目将按照“先讨论需求，再设计结构，再分阶段编码，再测试，再提交 GitHub”的流程推进。

## 项目目标

本项目计划实现一个简化但结构清晰的电梯模拟系统。

核心目标包括：

- 支持单台电梯运行；
- 支持楼层范围：地下 1 层到地上 34 层；
- 使用 -1 表示地下 1 层；
- 不存在 0 楼；
- 支持多个楼层请求；
- 采用“顺路优先，无顺路请求时最近优先”的调度策略；
- 模拟电梯上行、下行、开门、关门、停留等流程；
- 后续加入超重机制；
- 后续加入门阻挡检测与重新开门机制；
- 后续加入故障反应机制；
- 后续加入停电与恢复供电后的安全恢复机制；
- 后续加入管理员紧急暂停机制；
- 输出清晰的运行日志，便于观察状态变化。

## V1 功能

当前版本实现了一个最小可运行的单电梯状态机模拟程序。

已支持：

- 楼层范围：`-1` 到 `34`；
- `-1` 表示地下 1 层；
- 不存在 `0` 楼；
- 单台电梯；
- 多楼层请求；
- 请求去重；
- 顺路优先，无顺路请求时最近优先；
- 基础开门、停留、关门流程；
- 固定时间模拟；
- 命令行菜单交互；
- 清晰运行日志。

V1 暂不实现：

- 超重；
- 门阻挡；
- 故障；
- 停电恢复；
- 管理员暂停；
- 多电梯群控。

## V2 功能

当前版本继续加入了基础安全机制。

已支持：

- 最大载重：`1000kg`；
- 当前载重超过最大载重时，电梯保持开门，不能关门，不能运行；
- 载重恢复正常后，电梯会在安全条件满足时尝试关门；
- 门阻挡模拟；
- 门被阻挡时，关门会被阻止，并重新开门保持；
- 门阻挡解除后，电梯会在安全条件满足时尝试关门；
- 基础故障状态：门故障、电机故障、传感器故障、未知故障；
- 故障状态下停止普通调度；
- 管理员紧急暂停；
- 管理员暂停时停止普通调度，只能由管理员恢复；
- `Elevator_CanMove` 和 `Elevator_CanCloseDoor` 统一安全判断。

V2 暂不实现：

- 停电；
- 恢复供电后的安全恢复；
- 电梯停在两层之间；
- 多电梯群控；
- 图形界面。

## V3 功能

当前版本加入了停电与恢复供电机制。

已支持：

- 停电时立即停止普通调度；
- 停电后不响应普通楼层运行；
- 恢复供电后不直接继续原目标；
- 恢复供电后先进入安全恢复流程；
- 可以模拟电梯停在两层之间；
- 如果电梯停在两层之间，恢复时会低速移动到最近安全楼层；
- 到达安全楼层后执行开门、停留、关门流程；
- 恢复完成后再回到普通调度；
- 恢复完成后不会继续旧目标楼层，而是根据仍然存在的请求表重新调度；
- 状态面板显示断电、恢复中、楼层之间、安全楼层等信息。

V3 暂不实现：

- 真实物理位置；
- 真实传感器；
- 多种恢复路径；
- 维修人员流程；
- 多电梯群控。

当前版本让调度逻辑进一步接近真实电梯。

## V5 前置设计

V5 不会直接把图形界面和电梯核心逻辑混在一起。

项目会先保持这样的分层：

- `include/elevator.h` 定义电梯核心数据结构、事件入口和状态快照；
- `src/elevator.c` 负责电梯状态机、调度、安全机制和统计核心；
- `src/elevator_request.c` 负责请求表、外部/内部按钮入口、目标选择和请求统计记录；
- `src/elevator_door.c` 负责轿厢门、层门、门锁、开关门流程和门按钮；
- `src/elevator_power.c` 负责主电源、突然停电、备用电源救援和安全恢复流程；
- `src/elevator_safety.c` 负责载重、故障、管理员暂停、紧急呼叫和移动安全判断；
- `src/elevator_status.c` 负责状态、请求和统计信息输出；
- `src/elevator_names.c` 负责枚举值到可读文字的转换；
- `src/main.c` 暂时作为命令行界面，负责接收用户输入；
- 未来图形界面只负责显示状态、发送按钮事件，不直接修改电梯内部数据。

当前已新增：

- `ElevatorSnapshot`：用于保存当前电梯状态的只读快照；
- `Elevator_GetSnapshot`：把当前电梯状态复制到快照中，供未来界面层读取。
- `canMove` 和 `canCloseDoor`：让界面层知道当前是否允许移动或关门；
- `Elevator_PressHallUpButton`：外部上行按钮事件入口；
- `Elevator_PressHallDownButton`：外部下行按钮事件入口；
- `Elevator_PressCarFloorButton`：电梯内部楼层按钮事件入口。

这个设计的目的：

- 命令行界面和未来图形界面可以共用同一套电梯核心逻辑；
- GUI 可以通过快照显示楼层、方向、门状态、安全状态、按钮灯和统计数据；
- GUI 可以通过按钮事件入口发送用户操作，而不是直接修改请求表；
- 后续如果使用 Java、网页或其他界面，也不需要重写电梯状态机。

## V4.8 功能

当前版本加入了双层门安全模型。

真实电梯包含两类门：

- 轿厢门：电梯内部的门，跟随电梯移动；
- 层门：每一层电梯口的门，固定在楼层上。

已支持：

- 使用 `carDoor` 表示轿厢门；
- 使用 `landingDoors[TOTAL_FLOOR_COUNT]` 表示每一层层门；
- 使用 `landingDoorLocked[TOTAL_FLOOR_COUNT]` 表示每一层层门是否锁定；
- 使用 `isAlignedWithFloor` 表示电梯是否与当前楼层完成对齐；
- 所有层门初始化为关闭并锁定；
- 开门前必须确认电梯已与当前楼层对齐；
- 开门时只解锁并打开当前楼层层门，同时打开轿厢门；
- 关门时关闭当前楼层层门和轿厢门，然后重新锁定当前楼层层门；
- 电梯移动前必须满足：轿厢门关闭、所有层门关闭并锁定、电梯已对齐楼层；
- 状态面板显示轿厢门、当前层门、当前层门锁定状态、对齐状态和所有门安全状态。

V4.8 暂不实现：

- 独立的门位置百分比；
- 门锁传感器故障细分；
- 层门被外力打开的异常模拟；
- 开门按钮持续按下；
- 关门按钮；
- 紧急呼叫按钮。

后续可继续优化：

- 加入内部开门按钮持续按下时保持开门；
- 加入内部关门按钮，但仍受超重、门阻挡和门锁安全限制；
- 加入紧急呼叫按钮，优先级仅低于管理员和严重故障；
- 加入门锁故障，模拟层门无法锁定时禁止移动；
- 加入事件队列，用于模拟多个乘客几乎同时按按钮。

## V4.9 功能

当前版本加入了备用电源救援模型。

已支持：

- 区分人为关闭主电源和突然停电事故；
- 使用 `isMainPowerOn` 表示主电源状态；
- 使用 `isPowerFailure` 表示突然停电事故；
- 使用 `isBackupPowerAvailable` 表示备用电源是否可用；
- 使用 `isBackupPowerActive` 表示备用电源是否正在救援；
- 停电时保存停电前方向 `directionBeforePowerFailure`；
- 电梯停在两层之间时，记录救援楼层 `rescueFloor`；
- 如果突然停电且电梯在两层之间，备用电源会低速移动到救援楼层；
- 备用救援到达楼层后会打开轿厢门和当前层门，让乘客离开；
- 备用救援完成后，电梯停留在救援楼层，等待主电源恢复；
- 主电源关闭或突然停电期间，普通楼层按钮请求会被拒绝；
- 备用救援完成后，会清空旧的电梯内部目标楼层请求；
- 主电源恢复后，先进入安全恢复流程，再回到普通调度。

示例：

```text
电梯位于 15 和 16 层之间
停电前方向是上行
目标楼层原本是 24
突然停电后，备用电源不继续去 24 层
而是低速移动到 16 层，开门让乘客离开
之后停在 16 层等待主电源恢复
```

V4.9 暂不实现：

- 备用电源电量消耗；
- 备用电源故障；
- 更精确的两层之间物理位置；
- 自动计算上下相邻楼层的真实距离；
- 维修人员确认流程。

## V4.10 功能

已支持：

- 内部开门按钮按住状态；

## 安全优先级

当前系统按照以下优先级处理状态：

```text
1. 主电源关闭 / 突然停电
2. 备用电源救援 / 安全恢复
3. 管理员暂停
4. 故障状态
5. 紧急呼叫
6. 超重 / 门阻挡 / 开门按钮保持
7. 门锁、层门、轿厢门、楼层对齐检查
8. 内部楼层请求
9. 同方向外部请求
10. 反方向外部请求
11. 空闲回 1 楼候机
```

## 编译方式

在项目根目录执行：

```cmd
build.bat
```

双击 `build.bat` 也可以编译。脚本执行完成后会暂停，方便查看编译结果。

如果想手动输入完整编译命令，也可以执行：

```cmd
gcc -Wall -Wextra -Iinclude src/main.c src/elevator.c src/elevator_request.c src/elevator_door.c src/elevator_power.c src/elevator_safety.c src/elevator_status.c src/elevator_names.c -o %TEMP%\c_elevator_state_machine.exe
```

## 运行方式

先完成编译，然后执行：

```cmd
run.bat
```

双击 `run.bat` 也可以运行程序。

如果想手动运行编译产物，可以查看 `build.bat` 输出的实际路径，或查看：

```cmd
type %TEMP%\c_elevator_state_machine_path.txt
```
