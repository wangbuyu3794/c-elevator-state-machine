# C Elevator State Machine

## 项目简介

这是一个使用 C 语言实现的单电梯状态机模拟项目。

项目以“电梯运行控制”为场景，重点练习 C 语言中的结构体、枚举、函数拆分、头文件与源文件分离、状态机、事件处理、请求调度和异常安全机制。

本项目不是为了模拟真实商用电梯的全部复杂逻辑，而是通过一个接近现实的控制系统案例，训练如何把现实流程抽象成 C 语言程序。

## 项目目标

本项目计划实现一个简化但结构清晰的电梯模拟系统。

核心目标包括：

- 支持单台电梯运行；
- 支持楼层范围：地下 1 层到地上 34 层；
- 使用 -1 表示地下 1 层；
- 不存在 0 楼；
- 支持多个楼层请求；
- 采用“顺路优先，无顺路请求时最近优先”的调度策略；
- 模拟电梯上行、下行、开门、关门、停留等流程；
- 后续加入超重机制；
- 后续加入门阻挡检测与重新开门机制；
- 后续加入故障反应机制；
- 后续加入停电与恢复供电后的安全恢复机制；
- 后续加入管理员紧急暂停机制；
- 输出清晰的运行日志，便于观察状态变化。

## V1 功能

当前版本实现了一个最小可运行的单电梯状态机模拟程序。

已支持：

- 楼层范围：`-1` 到 `34`；
- `-1` 表示地下 1 层；
- 不存在 `0` 楼；
- 单台电梯；
- 多楼层请求；
- 请求去重；
- 顺路优先，无顺路请求时最近优先；
- 基础开门、停留、关门流程；
- 固定时间模拟；
- 命令行菜单交互；
- 清晰运行日志。

V1 暂不实现：

- 超重；
- 门阻挡；
- 故障；
- 停电恢复；
- 管理员暂停；
- 多电梯群控。

## V2 功能

当前版本继续加入了基础安全机制。

已支持：

- 最大载重：`1000kg`；
- 当前载重超过最大载重时，电梯保持开门，不能关门，不能运行；
- 载重恢复正常后，电梯会在安全条件满足时尝试关门；
- 门阻挡模拟；
- 门被阻挡时，关门会被阻止，并重新开门保持；
- 门阻挡解除后，电梯会在安全条件满足时尝试关门；
- 基础故障状态：门故障、电机故障、传感器故障、未知故障；
- 故障状态下停止普通调度；
- 管理员紧急暂停；
- 管理员暂停时停止普通调度，只能由管理员恢复；
- `Elevator_CanMove` 和 `Elevator_CanCloseDoor` 统一安全判断。

V2 暂不实现：

- 停电；
- 恢复供电后的安全恢复；
- 电梯停在两层之间；
- 多电梯群控；
- 图形界面。

## V3 功能

当前版本加入了停电与恢复供电机制。

已支持：

- 停电时立即停止普通调度；
- 停电后不响应普通楼层运行；
- 恢复供电后不直接继续原目标；
- 恢复供电后先进入安全恢复流程；
- 可以模拟电梯停在两层之间；
- 如果电梯停在两层之间，恢复时会低速移动到最近安全楼层；
- 到达安全楼层后执行开门、停留、关门流程；
- 恢复完成后再回到普通调度；
- 恢复完成后不会继续旧目标楼层，而是根据仍然存在的请求表重新调度；
- 状态面板显示断电、恢复中、楼层之间、安全楼层等信息。

V3 暂不实现：

- 真实物理位置；
- 真实传感器；
- 多种恢复路径；
- 维修人员流程；
- 多电梯群控。

## V4 功能

当前版本加入了基础请求统计功能。

已支持：

- 记录每个楼层请求加入请求表时的时间；
- 电梯真正服务该楼层时，计算该请求等待了多少秒；
- 等待时间口径为：从请求加入请求表，到电梯到达并开始服务该楼层；
- 统计已完成请求数量；
- 统计累计等待时间；
- 统计最长等待时间；
- 统计平均等待时间；
- 状态面板中显示统计信息；
- 菜单中可以单独查看统计信息。

V4 暂不实现：

- 外部上行/下行按钮的独立统计；
- 内部目标楼层按钮的独立统计；
- 随机乘客生成；
- 高峰期模拟；
- 多电梯群控；
- 更复杂的防饥饿调度算法。

## V4.5 功能

当前版本把请求模型改得更接近真实电梯。

已支持：

- 外部上行按钮请求；
- 外部下行按钮请求；
- 电梯内部楼层按钮请求；
- `-1` 层只能发起外部上行请求；
- `34` 层只能发起外部下行请求；
- `1` 到 `33` 层可以发起外部上行或下行请求；
- 电梯内部可以选择 `-1` 和 `1` 到 `34` 的目标楼层；
- 三类请求分别保存、分别显示；
- 三类请求分别统计等待时间；
- 旧的 `Elevator_AddRequest` 继续保留，并作为内部楼层按钮请求的兼容入口。

V4.5 暂不实现：

- 根据外部上行/下行按钮精确决定乘客目的地；
- 区分乘客进入电梯前后的完整流程；
- 内外按钮联动的完整乘客模型；
- 图形化内部/外部面板。

## V4.6 功能

当前版本让调度逻辑进一步接近真实电梯。

已支持：

- 电梯上行时，优先响应上方的内部楼层请求和外部上行请求；
- 电梯下行时，优先响应下方的内部楼层请求和外部下行请求；
- 到达楼层时，不再无条件清除该楼层所有请求；
- 到站服务会根据当前方向，只清除本次真正服务到的请求类型；
- 反方向外部请求会在电梯没有同方向请求时再被处理；
- 请求统计只统计本次真正完成服务的请求。

V4.6 暂不实现：

- 乘客进入电梯后自动生成内部目标楼层；
- 外部请求与乘客目的地的完整绑定；
- 图形化内部/外部面板；
- 多电梯群控。

## 编译方式

在项目根目录执行：

```powershell
gcc -Iinclude src/main.c src/elevator.c -o elevator.exe
```

## 运行方式

```powershell
.\elevator.exe
```

## 菜单说明

程序启动后可以选择：

- `1` 电梯内部楼层按钮；
- `2` 运行一步；
- `3` 一直运行到所有请求完成；
- `4` 查看当前电梯状态；
- `5` 设置当前载重；
- `6` 设置门是否被阻挡；
- `7` 设置故障；
- `8` 清除故障；
- `9` 管理员暂停；
- `10` 管理员恢复；
- `11` 停电；
- `12` 恢复供电；
- `13` 模拟电梯停在两层之间；
- `14` 手动运行恢复流程；
- `15` 查看请求统计；
- `16` 外部上行按钮；
- `17` 外部下行按钮；
- `0` 退出程序。
