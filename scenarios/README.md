# 验证场景

本目录保存可重复运行的命令行输入脚本，用于验证电梯状态机的主要行为。

## 运行方式

在项目根目录先编译：

```cmd
build.bat
```

运行单个场景：

```cmd
run.bat < scenarios\01_single_car_request.txt
```

运行全部场景：

```cmd
run_scenarios.bat
```

`run_scenarios.bat` 会自动编译程序，依次执行本目录下的 `.txt` 场景，并将日志写入 Windows 临时目录。

## 场景列表

### 01_single_car_request.txt

验证内部楼层按钮请求。

观察点：

- 5 楼内部请求被接受；
- 电梯从 1 楼上行到 5 楼；
- 到达后完成开门、停留、关门流程；
- 请求完成后电梯回到空闲状态。

### 02_multiple_direction_requests.txt

验证多方向请求调度。

观察点：

- 6 楼上行请求被接受；
- 8 楼内部请求被接受；
- 3 楼下行请求被接受；
- 电梯从 1 楼出发时优先处理上行任务；
- 电梯上行途中服务 6 楼上行请求；
- 3 楼下行请求等待电梯反向后再处理。

### 03_overload_blocks_run.txt

验证超重保护。

观察点：

- 当前载重设置为 1200kg；
- 电梯在超重状态下拒绝普通运行；
- 载重恢复到 500kg 后可以继续运行。

### 04_door_blocked_reopen.txt

验证门阻挡保护。

观察点：

- 门阻挡状态被设置；
- 电梯尝试关门时被安全条件阻止；
- 门阻挡解除后可以继续完成流程。

### 05_backup_power_rescue.txt

验证突然停电和备用电源救援。

观察点：

- 电梯开始向 24 楼运行；
- 模拟突然停电；
- 电梯被标记为停在两层之间；
- 备用电源可用；
- 备用救援将电梯移动到安全楼层并开门。

### 06_emergency_call_blocks_run.txt

验证紧急呼叫优先级。

观察点：

- 紧急呼叫被触发；
- 普通运行被阻止；
- 清除紧急呼叫后，电梯继续处理原有请求。
